  • HTTP — The way browsers ask a server for web pages and files.
  • HTTPS — A secure version of HTTP that protects data using encryption (TLS).
  • DNS — Turns website names (like google.com) into IP addresses so browsers can find the right server.
  • Load balancing — Spreads user traffic across multiple servers so no single server gets overloaded.
  • Caching — Saves copies of data so websites load faster and servers don’t have to work as hard.
  • Reverse proxy — A middle server that sits in front of other servers to help manage traffic, security, and performance.
  • Serverless — Running small pieces of code without managing servers; they automatically scale when needed.

Static vs Dynamic

Static files are simple files like HTML, images, and CSS that are sent directly to the user as they are. Dynamic content is created on the spot by an application using code, and it often pulls data from a database (for example: PHP, Node.js, Python, or Java backends).
